ENVIRONNEMENTS DE PROGRAMMATION

1. Linux/Windows

Les systèmes d'exploitation Linux (comme Ubuntu) et Windows reposent sur des architectures et des usages différents, ce qui les rend adaptés à des contextes distincts, notamment dans le domaine de la robotique et du développement logiciel.

a) Linux

Linux est un système open-source, ce qui signifie que son code source est librement accessible, modifiable et redistribuable. Pouvoir modifier son code source permet une grande flexibilité. Cet environnement est très présent dans la robotique, l'informatique embarquée, les serveurs ou encore dans les environnements de développement avancés. Son utilisation est notamment appréciée pour sa stabilité, sa compatibilité avec de nombreux outils scientifiques, en particulier pour ROS (Robot Operating System)

Les distributions Linux s'appuient sur des gestionnaires de paquets (comme APT sur Ubuntu), qui permettent d'installer, mettre à jour et supprimer des logiciels de manière centralisée, sécurisée et automatisée.

Linux met fortement en avant le terminal et les outils en ligne de commande, qui offrent un contrôle précis du système et sont très appréciés dans les domaines du développement, de la recherche et de l'automatisation. Il est également nativement compatible avec des langages tels que Python, C/C++ ou Bash, et avec des outils comme Git ou Docker.

Linux est reconnu pour sa robustesse, sa faible exposition aux virus et sa capacité à fonctionner de manière stable sur de longues durées, ce qui explique sa présence dans les serveurs, les infrastructures critiques ou les robots industriels

Dans la documentation pour contrôler le robot via ROS, il nous ait conseillé d'utiliser Ubuntu 20.04.

Ubuntu est une distribution Linux (un système d'exploitation basé sur Linux). La version 20.04 est une LTS (Long Term System), ce qui veut dire qu'elle est stable, fiable et maintenue longtemps. Elle est souvent utilisée en robotique, informatique scientifique, développement, et notamment pour ROS (Robot Operating System) car elle est très compatible.

b) Windows

Contrairement à Linux, Windows est un système propriétaire, développé et contrôlé par Microsoft. Son fonctionnement est moins modifiable ce qui offre une ouverture au grand public et une facilité d'utilisation.
Il est présent dans les usages des jeux vidéo, du multimédia et de la bureautique. Il est conçu pour offrir une interface intuitive.

Windows repose majoritairement sur des installateurs individuels téléchargés sur Internet ou via le Microsoft Store. Cela offre une plus grande simplicité pour l'utilisateur moyen, mais un contrôle moins fin pour les développeurs.

Windows privilégie une interface graphique conviviale, avec moins d'accent sur le terminal. Cependant, de nombreux outils robotiques ou scientifiques y sont moins bien supportés ou nécessitent des couches de compatibilité.
Windows offre également un bon niveau de sécurité, mais étant le système le plus répandu chez les particuliers, il est davantage ciblé par les logiciels malveillants. Il nécessite aussi des mises à jour fréquentes, parfois intrusives.

2. Environnements de simulation

Si l'accès physique au robot n'est pas disponible il est possible de passer par la simulation. Pour cela, différents outils sont disponibles :

  • NiryoStudio
  • RViz
  • Gazebo

La simulation par NiryoStudio, nécessite de payer une « extension ».

a) RViz

RViz est un outil de visualisation 3D pour ROS. Il permet d'afficher en temps réel :

  • Des robots
  • Leurs capteurs (caméra, capteurs IR, etc.)
  • Leurs trajectoires
  • Leurs cartes

En bref, c'est un logiciel pour voir ce que voit le robot et ce qu'il fait dans ROS.

b) Gazebo

Gazebo est un simulateur 3D de robots. Il permet de créer un environnement virtuel dans lequel un robot peut :

  • Se déplacer
  • Percevoir (capteurs simulés)
  • Interagir avec des objets

En bref, c'est une salle d'expérimentation virtuelle.

Remarque : dans notre cas nous avons fait un dual-boot qui nous permet d'utiliser NiryoStudio en étant dans l'environnement Windows et RViz et Gazebo dans un environnement Linux.
